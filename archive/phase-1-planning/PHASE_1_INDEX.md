## Key Decisions Made (Phase 0)

‚úÖ **Technology Stack**:
- Go 1.21+ (modern, performant)
- Bubble Tea + Lip Gloss (TUI framework)
- ledongthuc/pdf (PDF parsing)
- go-termimg (image rendering, Phase 3+)

‚úÖ **Architecture**:
- 3-pane layout (metadata, viewer, search)
- LRU cache for page performance
- Thread-safe operations
- Dark mode by default

‚úÖ **Development Approach**:
- Test-driven development (Phase 1.2+)
- Performance profiling (Phase 1.2+)
- Documentation-first (Phase 1.6)
- Milestone-based delivery

## Important Notes

### üéØ Goals for Phase 1

1. **Functional MVP**: Working PDF reader
2. **Quality**: 80%+ test coverage
3. **Performance**: Meet targets (relaxed from Phase 0)
4. **Stability**: No crashes, handles edge cases
5. **Documentation**: Complete and clear
6. **Foundation**: Ready for Phase 2

### ‚ö†Ô∏è Phase 1 Realities

- Performance targets are **relaxed** from Phase 0:
  - Cold start: <200ms (was <100ms)
  - Memory: <100MB (was <50MB)
  - This gives flexibility for MVP quality

- Focus is on **correctness first, optimization second**
  - We'll measure in 1.2
  - Optimize in 1.4
  - Only if needed

- Some features are **Phase 2+**:
  - Table of contents
  - Fuzzy search
  - Bookmarks
  - Better layout preservation
  - Image rendering (Phase 3)
